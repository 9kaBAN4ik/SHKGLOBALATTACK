# 📖 Руководство: Как получить координаты деревень для Discord бота

## Проблема

Discord бот будет работать на Linux сервере, где:
- ❌ Невозможно запустить игру
- ❌ Невозможно читать память процесса
- ❌ .NET Remoting блокируется сервером (работает только из игры)
- ❌ Игроки видят в игре только **villageID и villageName**, но НЕ видят координаты

## Решение

**Один раз экспортировать координаты на Windows компьютере, затем загрузить их на сервер!**

---

## Шаг 1: Экспорт координат (на Windows)

### Требования:
- Windows компьютер
- Установленный .NET SDK 4.8+
- Stronghold Kingdoms игра (декомпилированные файлы)

### Процесс:

1. **Убедитесь что у вас есть файл `.env`:**
   ```
   GAME_EMAIL=your_email@example.com
   GAME_PASSWORD=your_password
   shk_link=http://shk-w756.elb.fireflyops.com:80
   ```

2. **Запустите PowerShell скрипт:**
   ```powershell
   .\export_villages.ps1
   ```

3. **Программа выполнит:**
   - ✅ Авторизацию на сервере
   - ✅ Подключение к игровому серверу через .NET Remoting
   - ✅ Получение ВСЕХ ваших деревень с координатами
   - ✅ Сохранение в файл `villages.json`

4. **Результат:**
   ```json
   [
     {"id": 16107, "x": 423, "y": 582, "name": "Castle"},
     {"id": 74469, "x": 456, "y": 601, "name": "Farm1"},
     {"id": 83239, "x": 489, "y": 573, "name": "Farm2"}
   ]
   ```

### Что если возникла ошибка?

#### Ошибка: "❌ Ошибка авторизации"
- Проверьте email и password в `.env`
- Убедитесь что аккаунт активен

#### Ошибка: "❌ Не удалось подключиться к игровому серверу"
- Проверьте `shk_link` в `.env`
- Убедитесь что сервер доступен (Europa 10 = w756)

#### Ошибка: "⚠️ Не найдено ни одной деревни"
- Возможно, .NET Remoting вернул неожиданную структуру
- Попробуйте запустить игру и использовать memory reader (альтернатива)

## Шаг 2: Загрузка на сервер (Linux)

### 2.1. Скопируйте файл на сервер

**Через SCP:**
```bash
scp villages.json user@your-server:/path/to/discord-bot/
```

**Или через SFTP клиент** (FileZilla, WinSCP):
- Подключитесь к серверу
- Загрузите `villages.json` в папку с Discord ботом

### 2.2. Проверьте что файл на месте

```bash
ssh user@your-server
cd /path/to/discord-bot/
ls -lh villages.json
```

Должен отобразиться файл с размером >1KB.

---

## Шаг 3: Загрузка координат в Discord бота

### 3.1. Запустите команду в Discord

```
!loadvillages
```

### 3.2. Бот ответит

```
✅ Координаты деревень загружены!
Загружено в бота: 45 деревень
Отправлено в прокси: 45 деревень

Что дальше?
Теперь используйте команду:
!time all fromids: <id1;id2> targetid: <target_id>

Примеры загруженных деревень:
ID: 16107, X: 423, Y: 582
ID: 74469, X: 456, Y: 601
ID: 83239, X: 489, Y: 573
...
```

## Шаг 4: Использование бота

Теперь вы можете рассчитывать время атак:

```
!time all fromids: 16107;74469 targetid: 83239
```

Бот вернет таблицу с временем для всех множителей скорости:


## Обновление координат

### Когда нужно обновлять?

- ✅ При захвате новой деревни
- ✅ При потере деревни
- ✅ При перемещении деревни (если игра позволяет)

### Как обновить?

1. Запустите `export_villages.ps1` на Windows снова
2. Скопируйте новый `villages.json` на сервер
3. Выполните `!loadvillages` в Discord снова

Бот **перезапишет** старые координаты новыми.

---

## Альтернатива: Memory Reader (для продвинутых)

Если экспорт через .NET Remoting не работает, используйте **memory reader**:

1. Запустите игру на Windows
2. Используйте Cheat Engine для поиска адресов координат
3. Создайте C# программу для чтения памяти процесса
4. Экспортируйте в JSON

**Недостатки:**
- Требует запущенную игру
- Сложнее в настройке
- Адреса могут меняться после обновления игры

**Преимущества:**
- Работает всегда
- Можно автоматизировать обновление

## Часто задаваемые вопросы

### Q: Можно ли экспортировать деревни других игроков?

**A:** Нет, только свои. .NET Remoting возвращает только деревни вашего аккаунта.

### Q: Сколько времени занимает экспорт?

**A:** 5-10 секунд в зависимости от количества деревень.

### Q: Нужно ли держать игру запущенной?

**A:** Нет! Программа подключается напрямую к серверу через .NET Remoting.

### Q: Что если мой сервер не w756?

**A:** Измените `shk_link` в `.env` на ваш сервер (например: `http://shk-w123.elb.fireflyops.com:80`)

### Q: Работает ли на Linux?

**A:** Экспорт - только Windows. Discord бот - работает на Linux.
