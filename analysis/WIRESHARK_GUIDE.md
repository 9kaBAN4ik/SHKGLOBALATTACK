# Руководство по анализу трафика Stronghold Kingdoms

## Цель

Перехватить и проанализировать трафик между игрой и сервером, чтобы понять:
- Какие параметры отправляет игра в `LoginUserGuid`
- В каком формате данные
- Какие заголовки используются
- Есть ли дополнительные вызовы перед `LoginUserGuid`

## Шаг 1: Установка Wireshark

1. Скачать Wireshark: https://www.wireshark.org/download.html
2. Установить (выбрать все компоненты, включая WinPcap/Npcap)
3. Перезагрузить компьютер (если требуется)

## Шаг 2: Настройка захвата

1. Запустить Wireshark
2. Выбрать сетевой интерфейс (обычно Ethernet или Wi-Fi)
3. В поле фильтра ввести:
   ```
   tcp.port == 80 and ip.dst == 52.18.136.227
   ```
   (IP адрес сервера Europa 10 - может измениться, проверь через `ping shk-w756.elb.fireflyops.com`)

4. Нажать кнопку "Start capturing packets" (синяя акула)

## Шаг 3: Захват трафика игры

1. **НЕ ЗАКРЫВАТЬ Wireshark** - он должен работать в фоне
2. Запустить Stronghold Kingdoms
3. Войти в аккаунт (kara.bridges1991@comejoinuspro.org)
4. Выбрать мир Europa 10
5. Дождаться полной загрузки мира
6. Подождать 10-15 секунд
7. Вернуться в Wireshark
8. Нажать красную кнопку "Stop capturing"

## Шаг 4: Поиск нужных пакетов

1. В Wireshark найти пакеты с `POST /KingdomsRPC/Kingdoms.rem`
2. Это будут .NET Remoting вызовы
3. Найти пакет, который содержит `LoginUserGuid`

### Как найти LoginUserGuid:

1. В меню: Edit → Find Packet
2. Выбрать "Packet bytes"
3. Искать строку: `LoginUserGuid` (может быть в Unicode)
4. Или искать hex: `4C 6F 67 69 6E 55 73 65 72 47 75 69 64`

## Шаг 5: Экспорт данных

Когда найдешь нужный пакет:

1. Правой кнопкой на пакет → Follow → TCP Stream
2. Скопировать весь текст
3. Сохранить в файл `wireshark_capture.txt`

ИЛИ

1. Правой кнопкой на пакет → Export Packet Bytes
2. Сохранить как `request.bin`

## Шаг 6: Анализ данных

Запустить скрипт анализа:
```bash
py analyze_capture.py
```

Скрипт покажет:
- Размер запроса
- Сжат ли запрос
- Параметры вызова
- Заголовки HTTP

## Альтернативный метод: Fiddler

Если Wireshark сложен, можно использовать Fiddler:

1. Скачать: https://www.telerik.com/fiddler
2. Установить
3. Запустить Fiddler
4. Запустить игру
5. В Fiddler найти запросы к `shk-w756.elb.fireflyops.com`
6. Посмотреть содержимое запроса

## Что искать в захваченном трафике

### HTTP заголовки:
- `Content-Type`
- `Content-Encoding`
- `SOAPAction`
- Любые custom заголовки

### Тело запроса:
- Размер (должен быть ~200-300 байт)
- Сжатие (GZIP magic bytes: 1F 8B)
- После распаковки - бинарные данные .NET Remoting

### Параметры LoginUserGuid:
- `userName` - что именно передается?
- `userGuid` - совпадает с нашим?
- `sessionGuid` - совпадает с нашим?
- `needVillageData` - true или false?
- `versionID` - какое число?

## Ожидаемый результат

После анализа мы узнаем:
1. Точные параметры для `LoginUserGuid`
2. Нужны ли дополнительные вызовы
3. Правильный формат данных
4. Любые отличия от нашей реализации

Тогда мы сможем исправить `SimpleGameClient` и получить рабочее подключение!
