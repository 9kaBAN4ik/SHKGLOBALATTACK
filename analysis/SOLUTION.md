# Решение проблемы с координатами

## Проблема

Пользователи не могут вручную добавлять координаты через игру. Они будут писать только:
```
!time capx4 12345[1];67890[2] 99999
```

Боту нужно автоматически получать координаты деревень по их ID.

## Текущая ситуация

- ✅ XML-RPC авторизация работает (получаем UserGUID и SessionGUID)
- ❌ .NET Remoting подключение блокируется багом в CustomSinks.dll
- ❌ Сервер требует сжатие данных, но библиотека сжатия не работает

## Возможные решения

### Решение 1: Запуск игры в фоне (РЕКОМЕНДУЕТСЯ)

Запустить оригинальный клиент игры в фоновом режиме и использовать его для получения данных.

**План:**
1. Запустить StrongholdKingdoms.exe с параметрами автологина
2. Дождаться подключения к серверу
3. Использовать memory reading или IPC для получения данных из процесса
4. Извлечь координаты всех деревень
5. Сохранить в базу данных бота

**Преимущества:**
- Использует рабочий клиент с правильными DLL
- Не нужно исправлять CustomSinks
- Гарантированно работает

**Недостатки:**
- Требует запущенную игру
- Нужен memory reading или IPC

### Решение 2: Перехват трафика

Запустить игру и перехватить трафик через Wireshark/Fiddler.

**План:**
1. Запустить Wireshark
2. Запустить игру и войти в мир
3. Найти пакеты с координатами деревень
4. Извлечь данные
5. Создать скрипт для автоматического парсинга

**Преимущества:**
- Видим все данные
- Можем понять протокол

**Недостатки:**
- Данные сжаты (нужна распаковка)
- Одноразовое действие (нужно повторять)

### Решение 3: Исправление CustomSinks.dll

Декомпилировать и исправить баг в CustomSinks.dll.

**План:**
1. Декомпилировать CustomSinks.dll через ILSpy/dnSpy
2. Найти CompressedClientSink.ProcessResponse
3. Исправить код, чтобы не использовать stream.Length
4. Перекомпилировать DLL
5. Использовать исправленную версию

**Код проблемы (предположительно):**
```csharp
// НЕПРАВИЛЬНО:
long length = stream.Length; // ConnectStream не поддерживает Length

// ПРАВИЛЬНО:
// Читать поток до конца без получения длины
MemoryStream ms = new MemoryStream();
stream.CopyTo(ms);
long length = ms.Length;
```

### Решение 4: Создание базы данных координат

Создать веб-сервис, куда игроки будут загружать координаты один раз.

**План:**
1. Создать веб-форму для загрузки координат
2. Игроки экспортируют данные из игры (скриншот/текст)
3. Загружают на сайт
4. Бот использует эту базу данных

**Недостатки:**
- Требует ручной работы
- Координаты могут устареть

## Рекомендуемое решение: Исправление CustomSinks.dll

Это самое быстрое и надежное решение. Давайте сделаем это прямо сейчас.

### Шаги:

1. **Декомпилировать CustomSinks.dll**
   ```bash
   # Использовать ILSpy или dnSpy
   ```

2. **Найти проблемный код**
   - Класс: `CompressedSink.CompressedClientSink`
   - Метод: `ProcessResponse`
   - Строка: `stream.Length` или `stream.get_Length()`

3. **Исправить код**
   - Заменить прямое обращение к Length на копирование в MemoryStream
   - Или использовать другой способ чтения потока

4. **Перекомпилировать**
   ```bash
   csc /target:library /out:CustomSinks.dll CompressedClientSink.cs
   ```

5. **Заменить DLL**
   - Скопировать исправленную DLL в SHKGameConnector/bin/Debug/
   - Запустить снова

## Временное решение: Использование файла координат

Пока исправляем CustomSinks, можно использовать файл с координатами.

**Формат файла villages.json:**
```json
[
  {"villageId": 12345, "x": 100, "y": 200, "name": "Village1"},
  {"villageId": 67890, "x": 150, "y": 250, "name": "Village2"}
]
```

**Как получить:**
1. Запустить игру
2. Открыть карту
3. Вручную записать координаты нужных деревень
4. Сохранить в villages.json
5. Бот будет читать из этого файла

Это позволит боту работать прямо сейчас, пока мы исправляем подключение к серверу.
